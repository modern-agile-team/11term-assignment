## **To-Do List 시스템 요구사항 분석**

### **1. 기능 요구사항**

#### **1.1 할 일 생성 (Create)**
- 입력창에 텍스트 입력 후 `Enter` 키 또는 `+` 버튼 클릭
- 유효성 검증:
  - 빈 값인 경우 → 경고창/오류 메시지 표시
  - 유효한 값인 경우 → 새 항목 생성
- ID는 순차적으로 자동 증가 (1, 2, 3...)

#### **1.2 할 일 조회 (Read)**
- 생성된 모든 할 일 항목을 리스트로 표시
- 각 항목은 다음 요소 포함:
  - 체크박스 (완료 상태)
  - 할 일 텍스트
  - 수정 버튼 (연필 아이콘)
  - 삭제 버튼 (휴지통 아이콘)

#### **1.3 할 일 수정 (Update)**
- 연필 아이콘 클릭 시:
  - 해당 항목의 텍스트가 입력 가능한 상태로 변경
  - 연필 아이콘 → 완료 버튼으로 변경
- 수정 완료 트리거:
  - `Enter` 키 입력
  - 완료 버튼 클릭
- 유효성 검증:
  - 빈 값인 경우 → 경고창/오류 메시지 표시
  - 유효한 값인 경우 → 수정 반영
- **제약사항**: 완료된 항목(체크박스 체크됨)은 수정 불가

#### **1.4 할 일 삭제 (Delete)**
- 휴지통 아이콘 클릭 시 해당 항목 삭제

#### **1.5 완료 상태 토글**
- 체크박스 클릭 시:
  - 미완료 → 완료: 취소선 적용
  - 완료 → 미완료: 취소선 제거
- 완료된 항목은 수정 불가 (연필 버튼 비활성화 또는 숨김)

---

### **2. 기술 스택 구성**

#### **프론트엔드**
- HTML + CSS + JavaScript
- UI 이벤트 처리 (클릭, Enter 키)
- 서버와 HTTP 통신 (Fetch API/Axios)

#### **백엔드**
- REST API 구성
- CRUD 엔드포인트:
  - `POST /todos` - 생성
  - `GET /todos` - 전체 조회
  - `PUT /todos/{id}` - 수정
  - `DELETE /todos/{id}` - 삭제
  - `PATCH /todos/{id}/complete` - 완료 상태 토글
